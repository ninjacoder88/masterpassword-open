### FINDING-001 · CRITICAL — Use of Predictable Pseudo-Random Number Generator for Token Generation

| Field | Value |
|---|---|
| File | `/src/MasterPassword.BusinessLogic/TokenGenerator.cs` line 21 |
| OWASP | A02 - Cryptographic Failures |
| CWE | CWE-330 |

**Evidence:**

```
Random random = new Random();
```

**Description:** Use of 'System.Random' for generating tokens is insecure because it's a pseudo-random number generator and predictable. This makes tokens vulnerable to prediction attacks, potentially allowing attackers to forge valid tokens and gain unauthorized access.

**Attack Scenario:** An attacker who observes multiple generated tokens can reverse-engineer the internal state of the PRNG and predict future tokens. With predicted tokens, the attacker can impersonate any user and gain full access to their vault data, including encrypted notes and passwords.

**Remediation:** Replace 'System.Random' with 'System.Security.Cryptography.RandomNumberGenerator' or 'System.Security.Cryptography.RNGCryptoServiceProvider' for cryptographically secure random number generation.

### FINDING-004 · HIGH — Insecure Key Derivation Using Base64 Encoding and Truncation

| Field | Value |
|---|---|
| File | `/src/MasterPassword.BusinessLogic/KeyDeriver.cs` line 13 |
| OWASP | A02 - Cryptographic Failures |
| CWE | CWE-326 |

**Evidence:**

```
string encodedPassword = Convert.ToBase64String(Encoding.UTF8.GetBytes(password));
return encodedPassword.Length > 32 ? encodedPassword.Substring(0, 32) : encodedPassword.PadLeft(32);
```

**Description:** Insecure key derivation using Base64 encoding and truncation instead of a proper key derivation function (KDF) like PBKDF2 or Argon2. This approach does not provide sufficient entropy or resistance to brute-force attacks, making derived keys weak and vulnerable to cryptanalysis.

**Attack Scenario:** An attacker who obtains a user's master password can easily brute-force or perform rainbow table attacks against the derived key. This compromises the encryption of all vault data, allowing unauthorized access to encrypted notes and passwords.

**Remediation:** Replace the Base64 encoding approach with a proper key derivation function such as PBKDF2, bcrypt, or Argon2 with appropriate iterations and salt values.
